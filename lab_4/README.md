## **Ответы на вопросы для самопроверки:**

### **1. В чем разница между кодами C и S в таблице маршрутизации?**

- **C (Connected)**- маршруты к непосредственно подключенным сетям (интерфейсы маршрутизатора). Автоматически добавляются при настройке IP-адреса на интерфейсе.
- **S (Static)**- статические маршруты, настроенные администратором вручную командой`ip route`.

### **2. Какая команда используется для создания статического маршрута к сети 192.168.10.0 255.255.255.0, если следующий переход - 172.16.200.1?**

```bash
ip route 192.168.10.0 255.255.255.0 172.16.200.1
```

### **3. Для чего предназначен DHCP в клиентской сети?**

Для автоматического назначения IP-адресов и других сетевых параметров (маска подсети, шлюз по умолчанию, DNS-серверы) клиентским устройствам без ручной настройки.

### **4. Какой IP-адрес назначается рабочей станции после обновления при включенном DHCP?**

Первый свободный IP-адрес из пула DHCP, исключая адреса, указанные в`ip dhcp excluded-address`.

### **5. Какие еще параметры DHCP, не включенные в данное задание, можно задать на маршрутизаторе Cisco?**

- Доменное имя:`domain-name`
- Аренда адреса:`lease`(время аренды)
- NetBIOS серверы:`netbios-name-server`
- TFTP сервер:`bootfile`или`next-server`
- Опции DHCP:`option`

### **6. В чем цель использования протокола WEP в беспроводной сети?**

Для обеспечения конфиденциальности данных (шифрование) и контроля доступа (аутентификация) в беспроводной сети.

### **7. В чем значение ключа, который используется для защиты протокола WEP?**

Ключ WEP используется для:

- Шифрования данных между клиентом и точкой доступа
- Аутентификации клиентов в сети
- Может быть 64-битным (10 hex символов) или 128-битным (26 hex символов)

### **8. Является ли протокол WEP лучшим вариантом для защиты беспроводного доступа?**

**Нет.**WEP устарел и имеет серьезные уязвимости. Рекомендуется использовать WPA2 или WPA3 для надежной защиты.

### **9. Каково назначение статического NAT?**

Для постоянного сопоставления одного внутреннего частного IP-адреса одному внешнему публичному IP-адресу. Используется когда внешние устройства должны иметь постоянный доступ к внутреннему серверу.

### **10. Какая команда используется для назначения внутреннего интерфейса статического NAT?**

```bash
interface [interface_type] [number]
ip nat inside
```

### **11. Каково назначение NAT Overload? Какие команды необходимо ввести для его включения?**

**Назначение:**Трансляция многих внутренних частных адресов в один внешний публичный адрес с использованием разных портов (PAT - Port Address Translation).

**Команды для настройки:**

```bash
# Создание ACL для внутренних адресов
access-list 1 permit 192.168.1.0 0.0.0.255

# Настройка NAT Overload
ip nat inside source list 1 interface [outside_interface] overload

# Назначение интерфейсов
interface [inside_interface]
ip nat inside
exit

interface [outside_interface]
ip nat outside
exit
```
